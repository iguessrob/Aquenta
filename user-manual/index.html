A. Introduction

System Name

Aquenta: A Centralized Water Utility Consumption and Billing Management System

Purpose of the System

Aquenta is a centralized web-based system designed to streamline and modernize the water utility operations of a community-based cooperative. It integrates essential processes—from meter reading and consumption computation to billing, payment tracking, and report generation—into a single, unified platform.

The system is specifically developed to eliminate the inefficiencies of manual and spreadsheet-based workflows by providing automated and accurate processing of water consumption data. Through Aquenta, administrators can efficiently record meter readings, compute usage based on predefined tariff structures, generate billing statements, and monitor payment transactions and outstanding balances in real time.

In addition, Aquenta enhances operational transparency and data consistency by maintaining a centralized database of customer records, billing histories, and financial reports. It also supports decision-making by enabling the generation of comprehensive reports such as consumption summaries, arrears tracking, and aging receivables.

For customers, the system provides a dedicated portal that allows them to conveniently access their account information, review billing and payment history, and monitor their current balances without the need to visit the cooperative office.

Overall, Aquenta aims to improve efficiency, accuracy, transparency, and service delivery within the cooperative’s water utility management operations.

Target Users

  • Admin and Staff (Billing Officers): Operate daily transactions, maintain records, process bills and payments, and generate reports.
  • Customers (Concessionaires): Review account details, billing history, payment history, and outstanding balances.

B. System Access

Access via URL

  1. Open a web browser and type the official Aquenta URL: www.aquenta-coop.com.
  2. Wait for the landing page to load completely.
  3. Open the login page by cllcking the login or get started button from the landing page.

Supported Devices

  • Desktop computers (recommended for full admin operations)
  • Laptops
  • Mobile devices (for quick portal access and account checking)

Landing Page Walkthrough

  1. Review quick links and announcements, if present.
  2. Select Admin to open staff login, or Concessionaire to open customer login.
  3. Proceed to authentication to continue.

C. User Authentication

Admin Login Process

  1. Enter username and password on the admin login screen.
  2. Select Sign In.
  3. On success, the admin dashboard opens with module navigation.

Customer Login Process

  1. Open the customer login page from the landing page.
  2. Enter your customer credentials.
  3. Select Sign In to access the customer dashboard.

Input Validation and Error Handling

  • Required fields must not be empty.
  • Invalid credentials show an error and require correction before access is granted.
  • Users should confirm keyboard case and spacing when login fails.

D. Dashboard Overview

Admin Dashboard Overview

The admin dashboard is the control center for customer management, tariff setup, billing, payment encoding, and report generation.

Customer Portal Overview

The customer portal provides account visibility including outstanding balances, billing history, payment records, and customer profile details.

Navigation Menu Explanation

  • Customer Records: Add, edit, and view customer records.
  • Invoice: Review and print detailed charges, including consumption and total amount due.
  • Tariffs: Create, update, and remove rate entries.
  • Billing: Input meter readings and generate bills.
  • Payments: Record customer payments and update balances.
  • Reports: View and print operational and financial reports.

E. Module-by-Module Guide

1. Customer Management Module

Add Customer

  1. Open Customer Records and select Add Concessioner.
  2. Enter complete details: account name, address, meter details, and contact information.
  3. Select Add Concessioner.

Expected output: New customer record appears in the list and is available for billing.

Edit Customer

  1. Search for the customer in records.
  2. Select Edit and update needed fields.
  3. Select Save Changes.

Expected output: Updated customer information is saved and reflected in account views.

View Records

  1. Open Customer Records.
  2. Use filters or search by name/account number.
  3. Select the record for detailed view.

Expected output: Selected customer details and related records are displayed.

2. Tariffs Module

Create Tariff

  1. Open Tariffs and select Add Rate.
  2. Enter cubic meter rate and amount details.
  3. Select Add Rate.

Expected output: New tariff is available for billing computations.

Aquenta User Manual5

Update Tariff

  1. Select an existing tariff from the list.
  2. Edit the rate and relevant information.
  3. Select Save Changes.

Expected output: Modified tariff is applied for future billing runs.

Delete Tariff

  1. Locate the tariff entry.
  2. Select Delete and confirm the prompt.

Expected output: Tariff is removed from active listing.

3. Billing Module

Input Meter Readings

  1. Open Billing and select the customer account.
  2. Choose the desired billing period.
  3. Enter current meter reading.
  4. Validate previous reading reference before saving.

Automatic Consumption Computation

  1. After saving reading, the system computes consumption based on meter difference.
  2. The selected tariff is applied automatically.

F. User Workflows

Admin Workflow

  1. Access the system URL.
  2. Login as Admin.
  3. Navigate through dashboard modules.
  4. Add and manage customer records.
  5. Set or update tariffs.
  6. Input meter readings.
  7. Generate monthly bills.
  8. Record payments.
  9. Generate required reports.

  3. Login to Customer Portal.
  4. Check outstanding balances.
  5. Check billing history.
  6. View payment records.
  7. View account details.
  8. Logout securely.

H. Troubleshooting

Common issues and clear solutions
Issue Possible Cause Solution
Incorrect meter readings Typing error, wrong meter selected, or skipped validation. Re-open billing entry, verify previous and current reading values, then correct and regenerate bill.
Missing customer data Record was not saved completely or filtered out in listing views. Check customer records, clear active filters, and update or re-save incomplete profile fields.
Login errors Invalid credentials, incorrect role portal, or input formatting mistakes. Confirm role-specific login page, re-enter username/password carefully, and contact admin for credential reset if needed.
Support best practice: Capture the exact page, account number, date/time, and displayed error message before escalation.

I. Logout Process

Admin Logout Steps

  1. Complete current transaction and ensure all changes are saved.
  2. Select profile or account menu in the top bar.
  3. Select Logout.
  4. Confirm that the login page is displayed again.

Customer Logout Steps

  1. Review account and history screens as needed.
  2. Select the account menu in the customer portal.
  3. Select Logout.
  4. Confirm that portal access is closed and login screen appears.
Security reminder: Always logout after each session, especially on shared devices.
